### Pattern Coverage Gaps

| Gap | Affected Integration / Component | Severity | Recommendation |
|---|---|---|---|
| No retry/circuit-breaker/timeout policy found for any database connection (Postgres or SQL Server) | Web -> CatalogConnection/IdentityConnection; PublicApi -> CatalogConnection/IdentityConnection | High | Add Polly-based retry policies (EF Core `EnableRetryOnFailure`) with explicit max-retry-count and delay configuration; document as new NFRs |
| No retry/timeout policy found for inter-service HTTP calls (BlazorAdmin -> PublicApi, BlazorAdmin -> Web) | BlazorAdmin -> PublicApi (`apiBase`), BlazorAdmin -> Web (`webBase`) | Critical | Configure `HttpClient` with explicit timeout and a resilience pipeline (Polly / `Microsoft.Extensions.Http.Resilience`); both are synchronous calls with no declared fallback |
| No caching pattern evidence (no Redis, distributed cache, or `IMemoryCache` configuration found) despite `infra/abbreviations.json` listing a `cacheRedis: "redis-"` convention | Catalog browsing path (Web, PublicApi) | Medium | If catalog read traffic is significant, evaluate cache-aside for catalog item/brand/type lookups; if Redis was planned (per abbreviations file) but never provisioned, treat as a stale convention reference |
| No rate limiting found on PublicApi despite it being a public-facing REST API with Swagger UI | PublicApi | High | Add ASP.NET Core rate limiting middleware (.NET 8 built-in `Microsoft.AspNetCore.RateLimiting`) at minimum for unauthenticated endpoints |
| No health-check endpoint package (`Microsoft.Extensions.Diagnostics.HealthChecks` or equivalent) found in any of the 6 src `.csproj` files | Web, PublicApi | High | Add `AddHealthChecks()` with DB connectivity checks; required before any meaningful container orchestration health probing |
| JWT Bearer packages present in Web and PublicApi but no issuer/audience/signing-key configuration found anywhere | Web, PublicApi | High | Either complete the JWT configuration (if PublicApi is meant to be token-protected) or remove the unused package references to reduce attack-surface/confusion - confirm via `Program.cs` before deciding |
| Blazor WASM Authentication packages present in BlazorAdmin but no OIDC authority configured | BlazorAdmin | Medium | Confirm whether BlazorAdmin is meant to authenticate against ASP.NET Core Identity (Web) via an OIDC/OAuth2 flow; if so, authority/client configuration is missing from all 3 appsettings files |
